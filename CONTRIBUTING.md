# 🤝 贡献指南（如果你真的在乎的话 🤷）

## 🚀 怎么提交你的第一个 PR

1. 🍴 Fork 这个仓库
2. 🌿 创建一个你喜欢的分支（名字随便起，比如 `fix/我在凌晨三点写的代码` 🌙）
3. 💻 做你想做的事（也可以什么都不做 🛋️）
4. 📤 提交 PR
5. ✍️ 写一句提交信息（也可以不写，沉默是金 🤫）

## 📝 建议的提交信息格式

- `feat: 🆕 加了一坨新东西` 📦
- `fix: 🔧 修了一个不一定是 bug 的 bug` 🤔
- `refactor: 🔀 把代码搞得更乱了` 🌪️
- `chore: 🤷 我也不知道这算什么` ❓
- `feat!: 💣 我把 README 用二进制重写了` 🔥
- `wip: 🚧 还没想好要干嘛` 🤷‍♂️
- `docs: 📚 加了更多 emoji` 😤
- `test: 🧪 什么测试？` 🦄
- `style: 💅 给代码化了个妆` 💄
- `revert: ⏪ 回到过去` 🕰️
- `hotfix: 🔥 着火了！！！` 🚒
- `breaking: 💀 所有人的代码都坏了` 😈
- `merge: 🤝 大家都是朋友` 🫂

## 🚫 不建议做的事

- 🦠 上传病毒或恶意软件（这是犯罪 ⛓️）
- 📧 发送垃圾邮件（你不是尼日利亚王子 👑）
- 🔑 提交你的密码（请检查 .gitignore 🙈）
- 🧑‍🎓 过于认真（会破坏氛围 🎭）
- 🤡 在 PR 里写小说（那是 issue 的地盘 📖）
- 💀 删掉所有文件然后说 "我修了个 bug" （经典操作 🎬）

## 🎨 代码风格

随便 🤷。缩进？tabs vs spaces？无所谓 🤷‍♂️。

用 emoji 缩进也行 📐。用颜文字缩进也行 ( ͡° ͜ʖ ͡°)。

## 🧪 测试

什么测试？🦄 这里没有测试。

如果你写了测试，我们会很困惑 🤨。

如果你写了测试并且 coverage 100%，我们会怀疑你不是人类 🤖。
